Clarity Wiki User Guide

Welcome to Clarity Wiki v1, your streamlined solution for organizing notes, managing multiple wikis, and enhancing your productivity through a user-friendly interface. This guide will walk you through the installation process, explore the application's features, highlight essential keyboard shortcuts, and provide practical usage tips.

Essentially, Clarity Wiki helps you organise your notes into different 'wikis'. You might have a subject of study, which you need multiple but related notes on. Your subject can be a new wiki, and you can make notes on various topics in that wiki. You can take your notes in text and markdown, using the markdown syntax features; and you can link to ohter notes in your wiki by simply typing the [[related-note-name]] - if that note exists, you can simply click on the link to open it up. This is useful for keeping notes on multiple points. For instance, as a lawyer, you might want to make notes regarding a certain issue: you might then keep a central note for organising everything, and separate notes based on witnesses, or documents, or other authorities etc., relevant to the issue. With the [[link]] functionality, you can quickly draw links between multiple moving parts...

1. Installation Guide

a. Downloading Clarity Wiki

Obtain the Application Bundle:

- Download the latest **Clarity Wiki v1** `.zip` file from the official distribution source or the provided download link.

Extract the .zip File:

- Locate the downloaded `.zip` file in your `Downloads` folder.
- Double-click the `.zip` file to extract its contents. This will create a `Clarity Wiki.app` bundle.

Move to Applications Folder:

- Drag and drop the `Clarity Wiki.app` into your `Applications` folder for easy access and to ensure proper system integration.

b. Bypassing macOS Gatekeeper

macOS includes a security feature called Gatekeeper that restricts the installation of applications from unidentified developers. To run Clarity Wiki v1, you may need to bypass Gatekeeper as follows:

Attempt to Open Clarity Wiki:

- Navigate to the `Applications` folder.
- Double-click on `Clarity Wiki.app` to launch the application.

Encounter Gatekeeper Warning:

- If you receive a warning stating that the app cannot be opened because it is from an unidentified developer, proceed to the next steps.

Access Security Settings:

- Open **System Preferences** by clicking on the Apple logo in the top-left corner and selecting `System Preferences`.
- Click on `Security & Privacy`.
- In the `General` tab, locate the message about `Clarity Wiki.app` being blocked.

Allow the App to Open:

- Click the `Open Anyway` button next to the warning message.
- A confirmation dialog will appear; click `Open` to grant permission.
- **Clarity Wiki** will now launch and be accessible in the future without additional prompts.

Note: Only bypass Gatekeeper for applications you trust. Ensure that you have downloaded Clarity Wiki v1 from a reputable source to maintain system security.

    • Assign tags to notes for easy categorization and retrieval. Tags can be added or edited through the tags input field.

  • Multiple Export Formats:
    • Export your notes in both HTML and PDF formats, preserving the formatting and structure of your content.

    • Apply consistent styling to exported documents, including font preferences like Charter or Palatino, to maintain a professional appearance.

  • Intra-Wiki Search:

    • Perform searches across all wikis to locate notes regardless of their categorization.
  • Tag-Based Filtering:

g. Backlinks Management

  • Link Tracking:

3. Keyboard Shortcuts

Enhance your productivity by utilizing the following keyboard shortcuts within Clarity Wiki v1:

  • Bold Formatting:

    • Mac: Command + B --- Action: Wraps the selected text in double asterisks (**) to apply bold formatting.

  • Italic Formatting:

    • Mac: Command + I --- Action: Wraps the selected text in single asterisks (*) to apply italic formatting.

  • Delete Note:

    • Mac: Command + D --- Action: Deletes the currently open note after confirmation.

  • Create New Note:

    • Mac: Command + N--- Action: Opens a dialog to create a new note within the selected wiki.

  • Open Note:

    • Mac: Command + O --- Action: Opens a file dialog to select and open an existing note.

  • Save Note:

    • Mac: Command + S--- Action: Saves the current note, including any changes made to tags or content.

  • Export Note:

    • Mac: Command + E--- Action: Opens the export dialog to save the current note as HTML or PDF.

Note: Keyboard shortcuts may vary based on system configurations and updates. Refer to the application's help section for the most accurate and up-to-date shortcuts.

5. Troubleshooting and Support

While Clarity Wiki v1 is designed for stability and ease of use, you may encounter occasional issues. Below are common problems and their solutions:

a. Application Won't Open After Installation

  • Gatekeeper Restrictions:
    • Ensure that you have followed the steps to bypass Gatekeeper correctly. Revisit the Installation Guide section to confirm.
  • Corrupted Download:
    • If the application fails to open, try re-downloading the .zip file and reinstalling the application.

b. Notes Are Not Saving Properly

  • Permissions Issue:
    • Verify that you have the necessary read and write permissions for the Clarity Wiki notes directory located at ~/Library/Application Support/ClarityWiki/notes/.
  • Disk Space:
    • Ensure that your system has sufficient disk space to save and manage your notes.

c. Exported PDFs Lack Proper Formatting

  • Font Availability:
    • Confirm that the fonts specified in the CSS (Charter, Palatino, etc.) are installed on your system. If not, install the required fonts or adjust the CSS to use available fonts.
  • Incomplete Export:
    • Ensure that the note content is fully loaded before initiating the PDF export. Wait for any loading indicators to complete.

d. Search Functionality Not Working as Expected

  • Indexing Delay:
    • Allow the application some time to index your notes, especially after creating or modifying a large number of notes.
  • Check Search Query:
    • Ensure that your search query is correctly spelled and matches the content within your notes.

e. Application Crashes or Freezes
