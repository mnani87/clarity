Introduction

1. Getting Started

Installation

Clarity Explorer is available as a macOS application bundled into a .app file and compressed into a ZIP archive for easy distribution. Please note that since the app is not code-signed or notarized, you may need to adjust your security settings to allow it to run.

Steps:

  1. Download the ZIP File:

    • Obtain the Clarity Explorer ZIP file from the provided download link.

    • Extract the Application:

    • Locate the downloaded ZIP file, usually in your Downloads folder.

    • Double-click the ZIP file to extract it. This will create the ClarityExplorer.app file.

    • Move the Application to the Applications Folder (Optional but Recommended):

    • Drag and drop the ClarityExplorer.app into your Applications folder for easy access.

Launching the Application

Since the application is not code-signed or notarized, attempting to open it may result in a security warning. Follow these steps to launch Clarity Explorer:

  1. First Attempt to Open the App:

    • Double-click ClarityExplorer.app to open it.

    • A dialog will appear stating:

      "ClarityExplorer.app" can't be opened because it is from an unidentified developer.

  2. Bypass the Security Warning:

    • Click OK to dismiss the dialog.

    • Open System Preferences by clicking on the Apple menu (ï£¿) and selecting System Preferences.

    • Navigate to Security & Privacy.

    • In the General tab, you will see a message at the bottom:

      "ClarityExplorer.app" was blocked from opening because it is not from an identified developer.

    • Click the Open Anyway button next to this message.

  3. Confirm Opening the App:

    • A new dialog will appear:

      "macOS cannot verify the developer of 'ClarityExplorer.app'. Are you sure you want to open it?"

    • Click Open to launch the application.

  4. Subsequent Launches:

    • After this initial process, you can open Clarity Explorer normally by double-clicking the app icon.

Alternative Method to Open the App

If you prefer using the context menu:

  1. Right-Click Method:

    • Control-click (or right-click) on ClarityExplorer.app.
    • Select Open from the context menu.
    • A dialog will appear similar to the previous one.

  2. Confirm Opening:

    • Click Open to launch the application.

2. User Interface Overview

When you launch Clarity Explorer, the main window is divided into three primary sections:

  • Left Sidebar (Projects List): Displays all your projects.
  • Center Panel (Files List): Shows files in the selected project.
  • Right Panel (Preview Area): Previews the content of the selected file.

Additionally, there are buttons and search bars for various actions:

  • Buttons:

    • Add Project
    • Add File to Project
    • Manage Tags for File
    • Delete Project

    • Delete File

    • Search Bars:

    • Project Search Bar: Searches files within the current project.

    • Explorer Search Bar: Searches files across all projects.

3. Projects Management

Adding a New Project

  1. Click the Add Project button located at the bottom of the left sidebar.
  2. An input dialog will appear prompting you to enter a project name.
  3. Type in the desired project name and click OK.
  4. The new project will appear in the Projects List.

Deleting a Project

  1. Select the project you wish to delete from the Projects List.
  2. Click the Delete Project button.
  3. A confirmation dialog will appear.
  4. Click Yes to confirm deletion.

Note: Deleting a project will remove it from Clarity Explorer but will not delete any files from your computer.

4. File Management

Adding Files to a Project

  1. Ensure you have a project selected in the Projects List.
  2. Click the Add File to Project button.
  3. A file dialog will open.
  4. Select one or more files you wish to add to the project and click Open.
  5. The selected files will appear in the Files List associated with the project.

Deleting a File from a Project

  1. Select the file you wish to remove from the Files List.
  2. Click the Delete File button.
  3. A confirmation dialog will appear.
  4. Click Yes to confirm removal.

Note: This action removes the file from the project in Clarity Explorer but does not delete the file from your computer.

5. Tag Management

Adding or Editing Tags for a File

  1. Select a file in the Files List.
  2. Click the Manage Tags for File button.
  3. An input dialog will appear showing the current tags associated with the file.
  4. Add new tags or edit existing ones by typing them into the input field, separated by commas.
    • Example: report, finance, Q3
  5. Click OK to save the changes.
  6. The file's display name in the Files List will update to reflect the new tags.

6. Searching Files

Search Within a Project

  1. Use the Project Search Bar located above the Files List.
  2. Type your search query.
    • The search is case-insensitive and matches against file names and tags.
  3. The Files List will update in real-time to show only the files that match your query.

Search Across All Projects

  1. Use the Explorer Search Bar located below the Project Search Bar.
  2. Type your search query.
  3. The Files List will display files from all projects that match your query.

7. Previewing Files

Clarity Explorer allows you to preview certain file types directly within the application.

Supported File Types

  • Plain Text Files: .txt, .md
  • PDF Files: .pdf
  • HTML Files: .html
  • Microsoft Word Documents: .docx
  • OpenDocument Text Files: .odt, .odf
  • Image Files: .jpg, .png, .gif

How to Preview a File

  1. Select a file from the Files List.
  2. The content will automatically display in the Preview Area.
    • For text-based files, the content will appear as text.
    • For images, the image will be displayed.
    • For unsupported file types, a message will indicate that preview is not available.

8. Opening Files with Default Applications

To open a file using the default application associated with its file type:

  1. Double-click on the file in the Files List.
  2. The file will open using the default program on your macOS system.
    • For example, a .pdf file might open in Preview, and a .docx file might open in Microsoft Word.

10. Handling Missing Files

If a file cannot be found at its recorded location (e.g., it was moved or deleted), Clarity Explorer will prompt you when you attempt to preview or open the file.

Steps to Resolve Missing Files

  1. Warning Message:

    • A dialog will inform you that the file is missing.

    • Options:

    • Locate: Allows you to find the file at its new location.

      • Click Locate.
      • A file dialog will open. Navigate to the file's new location and select it.
      • Clarity Explorer will update the file path accordingly.
    • Remove from Project: Removes the file from the project.
      • Click Remove from Project.
      • The file will be removed from the Files List.
    • Cancel: Cancels the action.
      • Click Cancel to return without making changes.

11. Additional Tips

  • Consistent Tagging: Use consistent tags to improve searchability.
  • Project Organization: Group related files into projects for better organization.
  • Regular Backups: Since project data is stored in a JSON file, consider backing up your data periodically.
  • File Management: Keep files in their original locations to avoid missing file issues.

12. Troubleshooting

Application Not Launching

  • Issue: The application does not start or crashes immediately.
  • Solution:
    • Ensure the application was extracted properly from the ZIP file.
    • Confirm that you have followed the steps to bypass the security warning for unidentified developers.
    • Check for any error messages and consult the Contact and Support section if necessary.

File Not Previewing Correctly

  • Issue: The file content does not display in the Preview Area.
  • Solution:
    • Ensure the file is of a supported type.
    • Verify that the file exists at the specified location.

Cannot Open File with Default Application

  • Issue: Double-clicking a file does not open it.
  • Solution:
    • Confirm that the file type has an associated default application.
    • Check for any error messages that may appear.

13. Conclusion

Clarity Explorer is designed to simplify the organization and management of your files through projects and tagging. By leveraging the application's features, you can efficiently categorize, search, and access your files.

For optimal use:

  • Utilize tags to enhance search capabilities.
  • Keep your files organized within projects.
  • Regularly update the application and dependencies.
  • Ensure you trust the source before bypassing security warnings.

Note: The projects and files data are stored in a JSON file located at:

~/Library/Application Support/ClarityExplorer/projects.json

Do not modify this file manually to prevent data corruption.
